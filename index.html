Shepherd: a runtime that makes agent execution forkable, replayable, and reversible

Shepherd wraps an agent run in a Git-like trace with copy-on-write snapshots, enabling meta-agents to observe, fork, replay, and revert entire execution histories.

Debugging and optimizing agents demands observability; this substrate provides the primitive for meta-agents to supervise, compare, and improve other agents.

For buildersIntegrate Shepherd with an existing agent to capture runs, then replay a failed interaction step-by-step to understand where the planner diverged.
